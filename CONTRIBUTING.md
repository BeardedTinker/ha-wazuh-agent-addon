# Contributing

Thanks for helping improve this project.

## Quick start

1. Fork the repo
2. Create a branch: `feat/<name>` or `fix/<name>`
3. Make changes
4. Run checks locally where possible (ShellCheck / formatting)
5. Open a PR

## What we accept

- bug fixes
- documentation improvements
- safer defaults / hardening improvements
- CI improvements
- HA “native experience” improvements

## What we usually avoid

- features that require custom HA integrations
- privileged/container-breaking behavior (unless clearly justified and optional)

## Style

- Keep changes minimal and readable
- Prefer deterministic behavior and explicit validation
- Do not log secrets (enrollment keys)
